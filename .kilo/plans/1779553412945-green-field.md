# Plan: Fix detail page — rating, UI, lógica, seed data

## 🔴 CRÍTICO — Bug #78: RateMaterial crash columna MaterialMateid

**Causa:** EF Core crea una shadow FK `MaterialMateid` porque la relación `MaterialRating → Material` no está configurada explícitamente.

**Fix:** En `InventoryConfiguration.cs:135`, agregar:
```csharp
builder.HasOne<Material>().WithMany().HasForeignKey(mr => mr.Mateid)
       .OnDelete(DeleteBehavior.Restrict);
```

## 🟡 Issues de lógica y UX en DetailPage

### 1. Actions checklist vacío
El catálogo `action_catalog` tiene datos (acatid=1..n) pero NO se registran acciones en `maintenance_action_detail` al guardar. El wizard actual solo guarda la orden base, no las acciones realizadas.

**Fix futuro:** Agregar Step de selección de acciones en el wizard.

### 2. Componentes con vida útil para alertas
Los componentes en `action_catalog` con `useful_life_days > 0` son los que generan alertas `COMPONENTE_POR_CADUCAR`. Revisar seed data y agregar componentes con vida útil.

### 3. Rating navega a página errónea
Tras el rating exitoso (cuando se arregle el bug #78), el `Load()` refresca datos. Si hay error en recarga, `ExecuteAsync` muestra error pero no navega. El problema reportado ("url faltand api") ya no debería ocurrir, confirmar tras fix.

### 4. Layout desordenado: cards + listas
Rediseñar secciones como cards con input arriba y lista debajo:
- **Acciones:** Picker + botón Agregar → CollectionView debajo (checkbox + nombre + botón eliminar ✕)
- **Consumir material:** Picker material + Entry cantidad + botón Consumir → CollectionView debajo (nombre, lote, costo unit, cantidad, costo total, estrellas)
- **Instalar componente:** Picker catálogo + botón Instalar → CollectionView debajo (nombre, km instalación, vencimiento, ✕)

### 5. Orden finalizada = solo lectura
Si `Status == "FI"`:
- Botón "Cerrar Orden" no visible
- Deshabilitar inputs: Picker acciones, material, componente
- Ocultar botones "Completar", "Asignar", "Guardar Diagnóstico"
- Mostrar solo listas + info + Exportar PDF

### 6. Exportar PDF solo en orden finalizada
`ExportPdf` botón visible solo si `Status == "FI"`.

### 7. Header: espacio en blanco y orden
Ajustar Grid header: eliminar margen superior, compactar layout vertical.

### 8. Diagnóstico en finalizada
El API devuelve `diagnosis` con `generalStatus`, `observations`, `vehicleOperative`, `futureRecommendations`. El MAUI solo muestra 1 campo. Completar bindings.

### 9. Información de aceite dentro de Info General
Mover bindings de aceite a la card de Información General. Si no hay datos, mostrar "No hay información".

### 10. Eliminar items de listas (Acciones, Materiales, Componentes)
Agregar endpoint API `DELETE /api/v1/maintenances/{id}/actions/{madeid}` y similares para remover items. O mejor: buffering local + guardado batch al cerrar.
