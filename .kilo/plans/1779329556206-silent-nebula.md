## Problemas detectados

### P1. La página de detalle no carga datos del vehículo / info general
**Causa raíz:** En `MaintenanceDetailViewModel.cs` línea 96:
```csharp
var detail = await _apiService.GetAsync<MaintenanceDetailResponse>(endpoint);
```
La API retorna `ApiResponse<MaintenanceResponse>` (objeto con `success`, `data`, `message`, `errors`). Pero `GetAsync<MaintenanceDetailResponse>` deserializa **todo el wrapper** como `MaintenanceDetailResponse`. Como `MaintenanceDetailResponse` no tiene propiedades que coincidan con `data`/`success`/etc al nivel superior, **todas las propiedades quedan en default (0, null)**.

**Fix:** Deserializar el wrapper y extraer `Data`:
```csharp
var raw = await _apiService.GetAsync<ApiResponse<MaintenanceDetailResponse>>(endpoint);
var detail = raw?.Data;
```

### P2. Mismatch de nombres de propiedades entre API y MAUI
Aún con el fix anterior, la API retorna:
- `actionDetails` (camelCase) → MAUI espera `Actions` (nombres diferentes, no solo casing)
- `oilBrand`, `oilViscositySae`, `showOilInNextMaintenance`, etc. → MAUI espera `OilInfo` anidado

**Fix 2a:** Agregar `[JsonPropertyName("actionDetails")]` a la propiedad `Actions` en `MaintenanceDetailResponse`.

**Fix 2b:** Reemplazar `OilInfo` anidado por propiedades planas en `MaintenanceDetailResponse`:
- Agregar `OilBrand`, `OilViscositySae`, `ShowOilInNextMaintenance` (match directo con API)
- Actualizar XAML para bindear directamente a `MaintenanceDetail.OilBrand` en vez de `MaintenanceDetail.OilInfo.OilBrand`

### P3. Componentes instalados no se muestran
**Causa raíz:** Mismo problema del wrapper (P1). Los componentes sí vienen en la API (`MaintenanceResponse.Components` → `ComponentItem`), pero nunca llegan al MAUI porque `detail` está vacío.

**Fix:** Se soluciona con el fix del P1.

### P4. Cerrar orden falla con "Error de conexión"
**Causa raíz:** La API espera un body `MaintenanceCloseRequest { IsEmergencyComplete: bool? }` pero el MAUI llama:
```csharp
await _apiService.PutAsync<object>(endpoint);
```
Sin enviar body. El API devuelve 400 Bad Request → `HttpRequestException` → catch muestra "Error de conexión".

**Fix:** Enviar body aunque sea vacío:
```csharp
await _apiService.PutAsync<object>(endpoint, new { });
```

### P5. Rating de material navega a URL incorrecta
**Causa raíz tentativa:** Tras el rating exitoso, se llama `await Load()` que intenta recargar datos del detalle. Si el deserializador falla (P1), `ExecuteAsync` atrapa la excepción y muestra error, pero el usuario reporta "me mando a pagina con url faltand api". 

Posibles causas:
- `DisplayPromptAsync` en .NET 10 Android podría tener un bug de navegación
- O hay una excepción no controlada en el flujo del rating

**Fix:** Agregar try-catch específico alrededor del bloque de rating para registrar la excepción. Además, separar el rating en un método aparte para aislar el flujo.
